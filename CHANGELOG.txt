VELMA - Knowledge Base Vectorizado para Agentes
================================================
Changelog — formato: [version] fecha — descripcion

--------------------------------------------------------------------------------
[0.1.0] 2026-03-28 — Initial release: sistema completo con tests y embeddings
--------------------------------------------------------------------------------

SISTEMA CORE
  + setup_kb.py        — Crea schema SQLite completo (issues_log, docs_index,
                         files_index, functions_index, reasoning_log) con FTS5
                         virtual tables y triggers de sincronización automática
  + indexer.py         — Indexa archivos .py/.js/.ts y documentación .md por
                         chunks de headers, con hash-caching para skip si no cambió
  + search.py          — Motor de búsqueda híbrida FTS5 + vectorial con
                         Reciprocal Rank Fusion (RRF, k=60) + panel web Flask
  + merge_knowledge.py — Sincronización local→shared con deduplicación por
                         fingerprint MD5 y similitud semántica (enrich/conflict)
  + kb_utils.py        — Utilidades: compute_hash, detect_chunk_type,
                         get_chunk_weight, cosine_similarity, encode_text/texts

EMBEDDINGS (activados en esta release)
  + Modelo: all-MiniLM-L6-v2 (384 dimensiones, ~22MB, corre local sin API)
  + Singleton get_embedding_model() — carga una sola vez por proceso
  + encode_text() / encode_texts() — generación individual y en batch
  + Embeddings generados en: docs_index, issues_log, files_index, functions_index
  + Búsqueda vectorial: search_vector_issues() + search_vector_docs()
  + Threshold de confianza: 0.50 (ajustable; 0.75 recomendado para KB >500 entradas)
